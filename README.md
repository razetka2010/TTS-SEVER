# ТТС Север (tts_sever)

Веб-приложение на **Django** для учёта и обработки заявок: создание запросов, личный список, техническая панель, аналитика и TV-доска для отображения на большом экране без входа в систему (по секретному фрагменту URL).

## Требования

- **Python** 3.10+ (проект совместим с Python **3.14**: при необходимости подключается патч в `manage.py` и `tts_sever/settings.py`).
- Зависимости указаны в [`requirements.txt`](requirements.txt) (основная — Django 4.2).

## Быстрый старт

### 1. Клонирование и виртуальное окружение

```powershell
cd путь\к\tts_sever
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

На Linux/macOS:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 2. Переменные окружения (опционально)

Скопируйте пример и при необходимости отредактируйте:

```powershell
Copy-Item .env.example .env
```

- **`TV_BOARD_SECRET`** — секрет для TV-доски (минимум **12 символов**). Если не задан при `DEBUG=True`, используется значение разработки из настроек (не используйте его в продакшене).

Файл `.env` читается из корня проекта; переменные, уже заданные в системе, имеют приоритет.

### 3. Миграции и суперпользователь

```powershell
python manage.py migrate
python manage.py createsuperuser
```

Суперпользователь нужен для входа в [**админку Django**](https://docs.djangoproject.com/en/stable/ref/contrib/admin/) и для первичной настройки пользователей и ролей.

### 4. Запуск сервера разработки

```powershell
python manage.py runserver
```

Откройте в браузере: **http://127.0.0.1:8000/** — корень перенаправляет на список «Мои заявки» (`/requests/my/`). Страница входа: **http://127.0.0.1:8000/login/**.

## Основные URL

| Путь | Описание |
|------|----------|
| `/` | Редирект на «Мои заявки» |
| `/login/`, `/logout/` | Вход и выход |
| `/admin/` | Админка Django |
| `/requests/create/` | Создание заявки |
| `/requests/my/` | Список заявок пользователя |
| `/requests/tech/` | Техническая панель |
| `/requests/analytics/` | Аналитика |
| `/requests/tv/<секрет>/` | TV-доска (секрет = значение `TV_BOARD_SECRET`) |

API polling для обновления данных на TV: **`GET /requests/api/state/`** (используется шаблоном доски).

## База данных и статика

- По умолчанию используется **SQLite**: файл **`db.sqlite3`** в корне проекта (создаётся после `migrate`).
- Статика в разработке отдаётся из каталога **`static/`**. Для продакшена выполните:

```powershell
python manage.py collectstatic
```

и настройте веб-сервер раздачу из **`STATIC_ROOT`** (`staticfiles/`).

## Роли пользователей

Модель пользователя расширена полем **роль** (`apps.accounts.User`): администратор, тех. администратор, пользователь. Назначение ролей удобно делать через `/admin/`.

## Перед выкладкой в продакшен

В **`tts_sever/settings.py`** для боевого окружения обязательно:

- задать уникальный **`SECRET_KEY`** (не храните реальный ключ в репозитории — используйте переменные окружения или отдельный конфиг);
- установить **`DEBUG = False`**;
- заполнить **`ALLOWED_HOSTS`**;
- задать надёжный **`TV_BOARD_SECRET`** в `.env` или в окружении сервера.

При необходимости замените SQLite на PostgreSQL или другую СУБД через настройку **`DATABASES`**.

## Структура проекта (кратко)

- **`tts_sever/`** — настройки Django, `urls.py`, WSGI.
- **`apps/accounts/`** — пользователи и авторизация.
- **`apps/requests/`** — заявки, представления, фильтры, API состояния.
- **`templates/`** — HTML-шаблоны.
- **`static/`** — CSS, JS, изображения.

---

При ошибках импорта Django убедитесь, что активировано виртуальное окружение и выполнен `pip install -r requirements.txt`.
